================================================================================
NOTE OPERATIVE
================================================================================

  - Il modello judge `llama3.1:8b-instruct-q4_K_M` deve essere pullato
    sulla macchina che fa la run completa:
        ollama pull llama3.1:8b-instruct-q4_K_M

  - Per iterare velocemente dopo la prima run completa, usare il cache:
        python evaluation/tester.py --lang it --cache use
    Le invocazioni del chatbot vengono replayate da disco (gratis).
    Solo le LLM-call del judge pagano il costo. Risparmia ~2-3h per run.

  - La cache NON e invalidata automaticamente dai cambiamenti a:
      * SYSTEM_PROMPT in brain.py
      * indice Chroma (re-ingestion)
      * parametri di retrieval in config.py
    Dopo questi cambi: usare --cache refresh oppure cancellare
    evaluation/cache/.

  - I file ragas_metrics.csv di ogni run sono la fonte di verita per la
    relazione. Tenere sotto version control insieme a summary.md.
